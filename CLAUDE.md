# Claude Build Brief

Build a polished public landing page for WorldFork.

## Goal

Create a static marketing/documentation landing page. It should explain WorldFork, link to GitHub and docs, and show a visual model of branching timelines. It must not run WorldFork, connect to a backend, require API keys, or trigger simulations.

## Product Positioning

WorldFork is backend-first infrastructure for branching social simulations. It turns one scenario into multiple inspectable timelines with checkpointed ticks, agent reviews, job logs, lineage, and structured reports.

## Required Sections

- Hero with the WorldFork name and a short value proposition.
- Visual explanation of Scenario -> Big Bang -> Multiverse branches -> Reports.
- Brief "What it is" section.
- "How to set it up" section linking to the GitHub repo and docs.
- "Why it exists" or "About" section for public-facing context.
- Clear GitHub call to action.

## Constraints

- Static only.
- No backend calls.
- No simulation runner.
- No secret handling.
- Keep this repo independent from `Hilo-Hilo/WorldFork`.
- Preserve Astro unless there is a strong reason to change frameworks.
